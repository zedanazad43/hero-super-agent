# Copilot Instructions — Token-Efficient Development

You are a coding assistant for this repository. Follow these rules to MINIMIZE token
usage and avoid waste (this is a hard constraint from the project owner):

## Hard rules (token savings)
- NEVER generate code you have not been explicitly asked for. Do not "improve" or
  refactor unrequested files.
- Prefer small, targeted diffs. One logical change per PR/commit.
- Do NOT repeat the user's request back to them in your response.
- Do NOT add explanatory prose around code unless asked. Output the change only.
- Reuse existing utilities/helpers. Do not reinvent.
- For large files (>300 lines), edit only the changed region (±5 lines context).
  Do NOT rewrite the whole file.
- Skip tests/docs unless the task explicitly requires them.
- When fixing a bug, fix the root cause only — no surrounding "cleanup".

## Before you act
- Read ONLY the files needed for the change. Do not read the whole repo.
- Use narrow searches (symbol/semantic) instead of grepping everything.
- If a tool result is cached/recent, do not re-read it.

## PR / commit discipline
- Title must state the single change (verb + noun).
- Body: max 3 lines. No filler.
- Do not open PRs for changes already merged to main.

## Language
- Comments and identifiers: English. User chat may be Arabic — that is fine.

Violating token-saving rules wastes paid quota and will be rejected.
